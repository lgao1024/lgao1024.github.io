# 课题文件夹归档规范

该规范用于整理已经完成或已发表的课题项目文件，使数据、图表与稿件结构清晰，便于后续查阅、维护与复用。

---

# 1 项目文件夹命名规则

将所有项目文件整理到一个总文件夹中，命名格式为：

```
Project_Visual-Folder-System_2026
```

命名结构由三部分组成：

```
[类型]_[课题代号]_[年份]
```

- **类型（Project）**：文件夹类别，例如 `Project`
- **课题代号**：通常为课题关键词，通过 `-` 连接  
- **年份**：课题建立或开始的年份  

示例：

```
Project_DFT-Mechanism_2026
Project_CO2-Reduction_2024
```

---

# 2 项目目录结构

在 `Project` 目录下建立以下五个子文件夹：

```
01_Data
02_Plots
03_Figures
04_Manuscript
05_Submission
```

各目录用途如下。

---

# 2.1 数据文件（01_Data）

存放所有 **原始数据和处理后的数据文件**。

## 原始数据

所有原始数据需要压缩存储，并放入：

```
01_Data/_Raw-Data
```

命名示例：

```
DFT_Gibbs-Free-Energy_260201.zip
```

说明：

- 原始文件按照**测试类别分类打包**
- 需要时解压使用
- 使用完成后删除解压文件

## 数据处理文件

对于每一类测试，在 `01_Data` 中建立对应文件夹，例如：

```
01_Data
 ├─ Gibbs-Free-Energy
 ├─ Band-Structure
 └─ Adsorption
```

用于存放：

- 数据处理脚本
- 中间处理文件
- 导出数据
- 用于绘图和制表的数据

---

# 2.2 绘图文件（02_Plots）

存放所有 **绘图工程文件及导出图片**。

目录结构：

```
02_Plots
 └─ _Plot-Files
```

### 绘图源文件

可编辑绘图文件存放于：

```
02_Plots/_Plot-Files
```

命名示例：

```
DFT-Band-Structure_2602-v2.opju
```

### 导出图片

所有导出的图片放在：

```
02_Plots
```

命名示例：

```
Fig_Band-Structure_Raw_v2.png
```

---

# 2.3 论文图组（03_Figures）

存放论文中的 **组合图（Figure）文件**。

通常包括：

- `.pptx`（编辑文件）
- `.png`
- `.jpg`
- `.svg`
- `.pdf`

命名示例：

```
Figure1-v1.pptx
```

使用建议：

- **正文 Figure**  
  - 修改频繁  
  - 同一 `.pptx` 中可以保存多个版本  

- **SI 图组**  
  - 修改较少  
  - 可以集中放在一个 `.pptx` 文件中

---

# 2.4 稿件文件（04_Manuscript）

存放论文相关的全部文档：

- 正文
- 参考文献
- SI
- 表格
- 支持材料

## Draft 版本

初期稿件使用 `Draft` 命名：

```
Draft_DFT-Mechanism_2602-v0.9.docx
```

版本号规则：

- `v0.x` ：早期版本
- **小数点后**：小修改  
  - 语法  
  - 格式  
  - 局部调整  

- **小数点前**：大修改  
  - 大量内容改写  
  - 新增讨论  
  - 结构调整  

---

## Manuscript 版本

第一个正式投稿版本改为：

```
Manuscript_DFT-Mechanism_2603-v1.0.docx
```

---

## 参考文献文件

Endnote 文件：

```
Ref_DFT-Mechanism.enl
Ref_DFT-Mechanism.Data
```

---

## 支持文件

示例：

```
SI_DFT-Mechanism_2602-v0.5.docx
Tab_SI-Lattice-Params_2602-v1.xlsx
```

---

# 2.5 投稿文件（05_Submission）

存放 **投稿阶段的文件**。

常见文件：

```
Cover-Letter.docx
Manuscript.docx
Supporting-Information.docx
Manuscript_r1.docx
Response-Letter_r1.docx
Supporting-Information_r1.docx
```

---

## 投稿归档

当出现以下情况时：

- 修改审稿意见
- 更换投稿期刊

需要将当前投稿文件打包归档。

示例：

```
JACS_2601_Submission.zip
JACS_2602_Reversion-01.zip
```

存放于：

```
05_Submission/_Archieve
```

---

## 稿件回存

在投稿周期结束后：

1. 将 **最终版本稿件复制一份** 到：

```
04_Manuscript
```

2. 不需要保存 `05_Submission` 中的其他版本到 `04_Manuscript`

3. 项目结束或重新投稿时，可开启新的投稿循环
